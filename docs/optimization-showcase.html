用了哪些优化

inpainter

bbox 局部修补

不再对整页 2048px 跑 LaMa,而是按 mask 连通域裁出每个文字块小图(dilate 合并相邻+留上下文)分别修补再贴回;碎框过多或覆盖过大自动回退整页。已覆盖 lama_aot / lama_large / lama_mpe。

0.93s → 0.36s · -61%
runtime / 显存

共享单份模型

模型池从「每图整份复制 Vec<Models>」改为单份 Arc<Models> 共享,整条 execute 走 &self,并发由信号量受控。避免高并发显存翻倍,并让翻译缓存可跨图共享。

架构 · 使 ③ 成为可能
translator

翻译缓存

按单条文本缓存译文(key = 译者链+配置的哈希 + 原文),命中即复用、跳过 LLM 请求。重复台词、角色名、重跑整批都免翻。

重复内容 → 0ms

主链路 · 热点位置

预处理 → upscaler → detector → ocr → textline-merge → translator ③ → mask-refinement → inpainter ① → compose → render → write
诚实说明: 上面的 ×3.2 含「重跑同一批图」的缓存红利——translator 因 ③ 全命中归零,贡献了大部分提速。首次翻译全新章节时 translator 仍会正常请求 DeepSeek(不命中)。对全新内容也持续生效的、最扎实的算法收益是 ① inpainter -61%;③ 缓存则在重跑、跨页重复台词/角色名时省时。
